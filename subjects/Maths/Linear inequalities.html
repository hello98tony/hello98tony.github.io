Introduction to Linear Inequalities

Linear inequalities are mathematical expressions comparing two quantities using inequality symbols. They form the foundation for optimization problems, decision-making, and various real-world applications.

Linear Inequality: An inequality that can be written in the form ax + b < 0, ax + b ≤ 0, ax + b > 0, or ax + b ≥ 0, where a and b are real numbers and a ≠ 0.

For two variables: ax + by < c, ax + by ≤ c, ax + by > c, or ax + by ≥ c

Basic Properties of Inequalities:

  • Addition/Subtraction: If a < b, then a ± c < b ± c
  • Multiplication by positive number: If a < b and c > 0, then ac < bc
  • Multiplication by negative number: If a < b and c < 0, then ac > bc (sign reverses!)
  • Division by positive number: If a < b and c > 0, then a/c < b/c
  • Division by negative number: If a < b and c < 0, then a/c > b/c (sign reverses!)
  • Transitivity: If a < b and b < c, then a < c
  • Trichotomy: For any two real numbers a and b, exactly one is true: a < b, a = b, or a > b

Algebraic Solutions of Linear Inequalities

Solving linear inequalities algebraically involves isolating the variable using properties of inequalities, with careful attention to sign reversal when multiplying/dividing by negative numbers.

Solution of Inequality: A value (or set of values) that makes the inequality true when substituted for the variable.

Solution Set: The set of all solutions of an inequality, often expressed using interval notation or set-builder notation.

For inequality ax + b < c:
Solution: x < (c - b)/a if a > 0, or x > (c - b)/a if a < 0

Key Point: When multiplying or dividing by a negative number, reverse the inequality sign!

Step-by-Step Solution Process

Example: Solving 3 - 2x ≤ 7

Step 1: Write the inequality: 3 - 2x ≤ 7

Step 2: Subtract 3 from both sides: -2x ≤ 7 - 3

Step 3: Simplify: -2x ≤ 4

Step 4: Divide both sides by -2 (reverse sign!): x ≥ 4/(-2)

Step 5: Simplify: x ≥ -2

Step 6: Solution set: {x ∈ ℝ : x ≥ -2} or in interval notation: [-2, ∞)

Graphical representation: On number line, closed circle at -2, shaded to the right

Special Cases and Important Rules:

  • When multiplying/dividing by negative number: REVERSE the inequality sign
  • Inequalities like x² < a² simplify to -a < x < a (for a > 0)
  • For inequalities involving absolute value: |x| < a ⇔ -a < x < a (for a > 0)
  • For inequalities involving absolute value: |x| > a ⇔ x < -a or x > a (for a > 0)
  • When solving rational inequalities, exclude values that make denominator zero
  • For quadratic inequalities: Find roots, then test values in each interval
  • Compound inequalities: Solve each part separately, then take intersection
  • When both sides are multiplied/divided by variable expression, consider positive and negative cases separately
Example: Quadratic Inequality

Solve: x² - 3x - 4 ≤ 0

Solution:

Step 1: Find roots: x² - 3x - 4 = 0 ⇒ (x-4)(x+1) = 0 ⇒ x = -1, 4

Step 2: Plot roots on number line, creating intervals: (-∞, -1), (-1, 4), (4, ∞)

Step 3: Test a value from each interval:

• Interval (-∞, -1): Test x = -2 ⇒ (-2)² - 3(-2) - 4 = 4 + 6 - 4 = 6 > 0

• Interval (-1, 4): Test x = 0 ⇒ 0² - 3(0) - 4 = -4 < 0 ✓

• Interval (4, ∞): Test x = 5 ⇒ 5² - 3(5) - 4 = 25 - 15 - 4 = 6 > 0

Step 4: We want x² - 3x - 4 ≤ 0 (non-positive), so choose intervals where expression ≤ 0

At x = -1 and x = 4, expression = 0 (included since ≤)

Step 5: Solution: -1 ≤ x ≤ 4, or in interval notation: [-1, 4]

JEE TIP

Inequality Solving Strategies

• For linear: Isolate variable, remember sign reversal with negative multiplication/division

• For quadratic: Factor, find roots, use sign chart or wavy curve method

• For rational: Find critical points (zeros and undefined points), test intervals

• For absolute value: Consider cases based on definition of absolute value

• For compound: Solve each inequality separately, then find intersection

• Always check endpoints for inclusion/exclusion in final answer

Graphical Method for Linear Inequalities

The graphical method represents linear inequalities in two variables as half-planes in the coordinate plane, with the boundary line separating the solution region from the non-solution region.

Graphical Representation: For inequality ax + by < c or ax + by > c:

• Draw boundary line ax + by = c (dashed for < or >, solid for ≤ or ≥)
• Test a point (usually origin if not on line) to determine which half-plane satisfies inequality
• Shade the half-plane that contains the solutions

Step-by-Step Graphical Method

  1. Convert to equation: Replace inequality sign with = to get boundary line equation
  2. Draw boundary line: Find intercepts or use slope-intercept form
    • x-intercept: Set y=0, solve for x
    • y-intercept: Set x=0, solve for y
  3. Line style: Solid line for ≤ or ≥ (boundary included), dashed line for < or > (boundary not included)
  4. Test point: Choose a point not on the line (usually (0,0) if not on line)
    • If test point satisfies inequality, shade that half-plane
    • If test point doesn't satisfy inequality, shade the opposite half-plane
  5. Shade region: Shade the half-plane containing all solutions

Key Points for Graphical Method:

  • The solution set of a linear inequality in two variables is always a half-plane
  • Boundary line: ax + by = c
  • Solid line: Boundary included in solution (≤ or ≥)
  • Dashed line: Boundary not included in solution (< or >)
  • Test point: Use (0,0) if it's not on the boundary line
  • If (0,0) is on the boundary line, use another point like (1,0) or (0,1)
  • Shading: The half-plane containing the test point if it satisfies the inequality
  • For vertical lines x = a: x < a is left half-plane, x > a is right half-plane
  • For horizontal lines y = b: y < b is lower half-plane, y > b is upper half-plane
  • Always label axes and indicate scale
Example: Graphical Solution of 2x + 3y ≤ 12

Step 1: Boundary line: 2x + 3y = 12

Step 2: Find intercepts:
• x-intercept: Set y=0 ⇒ 2x = 12 ⇒ x = 6 ⇒ Point (6, 0)
• y-intercept: Set x=0 ⇒ 3y = 12 ⇒ y = 4 ⇒ Point (0, 4)

Step 3: Draw line through (6,0) and (0,4). Since inequality is ≤, line is solid.

Step 4: Test point: Use (0,0) ⇒ 2(0) + 3(0) = 0 ≤ 12 ✓ (satisfies)

Step 5: Since (0,0) satisfies inequality, shade the half-plane containing (0,0).

Step 6: The solution region is the half-plane below and including the line 2x + 3y = 12.

Verification: Test point (3,1): 2(3) + 3(1) = 6 + 3 = 9 ≤ 12 ✓ (in shaded region)
Test point (4,3): 2(4) + 3(3) = 8 + 9 = 17 ≤ 12 ✗ (not in shaded region)

Systems of Linear Inequalities

A system of linear inequalities consists of two or more linear inequalities in the same variables. The solution is the intersection of the solution sets of all inequalities in the system.

System of Linear Inequalities: A set of two or more linear inequalities in the same variables.

Solution: Ordered pair (x, y) that satisfies ALL inequalities in the system simultaneously.

Feasible Region: The intersection of all solution half-planes, representing all solutions of the system.

System of Inequalities Visualizer

Solving Systems Graphically

  1. Graph each inequality: Treat each inequality separately
    • Graph boundary line (solid/dashed as appropriate)
    • Shade the solution half-plane for each inequality
  2. Find intersection: The feasible region is where ALL shaded regions overlap
  3. Identify feasible region: This is the solution set of the system
    • Bounded: Region is enclosed (polygon)
    • Unbounded: Region extends infinitely in at least one direction
    • Empty: No overlap (no solution)
  4. Find corner points: Vertices of the feasible region (where boundary lines intersect)

Properties of Systems of Inequalities:

  • The solution set of a system is the intersection of solution sets of individual inequalities
  • If the system has no feasible region (empty intersection), it has no solution
  • For a system of two variables, the feasible region is always a convex polygon (or unbounded convex region)
  • Corner points (vertices) of the feasible region are found by solving pairs of boundary line equations
  • A bounded feasible region always has a finite number of corner points
  • An unbounded feasible region may have a finite optimal value in linear programming if the objective function increases/decreases appropriately
  • Non-negativity constraints (x ≥ 0, y ≥ 0) are common in real-world problems and restrict the feasible region to the first quadrant
  • The feasible region is always a convex set: If two points are in the region, the entire line segment joining them is also in the region
Example: System of Inequalities

Solve graphically: x + 2y ≤ 10, 2x + y ≤ 10, x ≥ 0, y ≥ 0

Solution:

Step 1: Graph each inequality:

1. x + 2y ≤ 10: Boundary x + 2y = 10 ⇒ Points (10,0) and (0,5). Solid line, shade below.

2. 2x + y ≤ 10: Boundary 2x + y = 10 ⇒ Points (5,0) and (0,10). Solid line, shade below.

3. x ≥ 0: Vertical line x = 0 (y-axis), solid, shade to the right.

4. y ≥ 0: Horizontal line y = 0 (x-axis), solid, shade above.

Step 2: Feasible region is the intersection of all shaded regions (quadrant I, below both lines).

Step 3: Find corner points:

• Intersection of x-axis (y=0) and 2x+y=10: 2x=10 ⇒ x=5 ⇒ Point A(5,0)

• Intersection of y-axis (x=0) and x+2y=10: 2y=10 ⇒ y=5 ⇒ Point B(0,5)

• Intersection of x+2y=10 and 2x+y=10:
Solve: x+2y=10 ...(1), 2x+y=10 ...(2)
From (1): x=10-2y. Substitute in (2): 2(10-2y)+y=10 ⇒ 20-4y+y=10 ⇒ -3y=-10 ⇒ y=10/3
Then x=10-2(10/3)=10-20/3=10/3
Point C(10/3, 10/3)

• Origin: O(0,0) (intersection of x≥0 and y≥0)

Step 4: Feasible region is quadrilateral with vertices O(0,0), A(5,0), C(10/3, 10/3), B(0,5).

All points in this quadrilateral (including boundaries) satisfy all four inequalities.

Linear Programming Problems

Linear Programming (LP) involves optimizing (maximizing or minimizing) a linear objective function subject to linear inequality constraints. It has wide applications in business, economics, and engineering.

Linear Programming Problem: Consists of:

1. Objective function: Linear function to maximize or minimize (z = ax + by)
2. Constraints: System of linear inequalities
3. Non-negativity restrictions: x ≥ 0, y ≥ 0 (usually)

Standard Form:
Maximize/Minimize: z = ax + by
Subject to: a₁x + b₁y ≤ c₁, a₂x + b₂y ≤ c₂, ..., x ≥ 0, y ≥ 0

Fundamental Theorem of Linear Programming

For a linear programming problem:

  1. If there is an optimal solution, it occurs at a corner point (vertex) of the feasible region.
  2. If the optimal solution exists, there is at least one corner point that is optimal.
  3. If two adjacent corner points are optimal, then every point on the line segment joining them is also optimal (multiple optimal solutions).
  4. If the feasible region is bounded, the optimal solution always exists.
  5. If the feasible region is unbounded, the optimal solution may or may not exist.

Solving Linear Programming Problems Graphically

  1. Formulate the problem: Identify decision variables, objective function, and constraints
  2. Graph the constraints: Plot each inequality to find feasible region
  3. Identify feasible region: Determine the intersection of all constraints
  4. Find corner points: Determine coordinates of all vertices of feasible region
  5. Evaluate objective function: Calculate z = ax + by at each corner point
  6. Determine optimal solution:
    • For maximization: Choose point with largest z-value
    • For minimization: Choose point with smallest z-value
  7. Check for multiple solutions: If two corner points give same optimal value, all points on the line segment between them are optimal
  8. Interpret the solution: State optimal values of variables and optimal objective function value

Example: Linear Programming Problem

A company manufactures two products, A and B. Each unit of A requires 2 hours of machine time and 3 hours of labor. Each unit of B requires 3 hours of machine time and 1 hour of labor. The company has at most 60 hours of machine time and 48 hours of labor available per day. Profit is ₹50 per unit of A and ₹40 per unit of B. How many units of each should be produced daily to maximize profit?

Solution:

Step 1: Define variables: Let x = units of A, y = units of B

Step 2: Objective function: Maximize z = 50x + 40y (profit)

Step 3: Constraints:
• Machine time: 2x + 3y ≤ 60
• Labor time: 3x + y ≤ 48
• Non-negativity: x ≥ 0, y ≥ 0

Step 4: Graph constraints and find feasible region (bounded quadrilateral in first quadrant).

Step 5: Corner points:
• O(0,0): Intersection of x=0 and y=0
• A(16,0): Intersection of y=0 and 3x+y=48 ⇒ 3x=48 ⇒ x=16
• B(0,20): Intersection of x=0 and 2x+3y=60 ⇒ 3y=60 ⇒ y=20
• C(12,12): Intersection of 2x+3y=60 and 3x+y=48
Solve: 2x+3y=60 ...(1), 3x+y=48 ...(2)
From (2): y=48-3x. Substitute in (1): 2x+3(48-3x)=60 ⇒ 2x+144-9x=60 ⇒ -7x=-84 ⇒ x=12
Then y=48-3(12)=48-36=12

Step 6: Evaluate z = 50x + 40y at each corner point:
• O(0,0): z = 50(0)+40(0) = 0
• A(16,0): z = 50(16)+40(0) = 800
• B(0,20): z = 50(0)+40(20) = 800
• C(12,12): z = 50(12)+40(12) = 600+480 = 1080

Step 7: Maximum profit is ₹1080 at point C(12,12).

Conclusion: Produce 12 units of A and 12 units of B daily for maximum profit of ₹1080.

Note: Points A and B give same profit (₹800), but point C gives higher profit.

Example: Diet Problem Application

A nutritionist is planning a diet that includes two foods, F1 and F2. Each unit of F1 costs ₹20 and contains 3 units of vitamins, 4 units of minerals, and 1 unit of protein. Each unit of F2 costs ₹30 and contains 2 units of vitamins, 2 units of minerals, and 3 units of protein. The diet must provide at least 60 units of vitamins, 75 units of minerals, and 60 units of protein. How many units of each food should be used to meet requirements at minimum cost?

Solution:

Step 1: Define variables: Let x = units of F1, y = units of F2

Step 2: Objective function: Minimize cost C = 20x + 30y

Step 3: Constraints:
• Vitamins: 3x + 2y ≥ 60
• Minerals: 4x + 2y ≥ 75
• Protein: x + 3y ≥ 60
• Non-negativity: x ≥ 0, y ≥ 0

Step 4: Graph constraints (all ≥, so feasible region is unbounded above).
Find corner points by solving systems of equations:
• Intersection of 3x+2y=60 and 4x+2y=75: Subtract: x=15, then y=(60-45)/2=7.5 ⇒ A(15, 7.5)
• Intersection of 3x+2y=60 and x+3y=60: Solve ⇒ x=12, y=16 ⇒ B(12, 16)
• Intersection of 4x+2y=75 and x+3y=60: Solve ⇒ x=16.5, y=14.5 ⇒ C(16.5, 14.5)

Step 5: Evaluate C = 20x + 30y at corner points:
• A(15, 7.5): C = 20(15) + 30(7.5) = 300 + 225 = 525
• B(12, 16): C = 20(12) + 30(16) = 240 + 480 = 720
• C(16.5, 14.5): C = 20(16.5) + 30(14.5) = 330 + 435 = 765

Step 6: Minimum cost is ₹525 at point A(15, 7.5).

Step 7: Check if feasible region is bounded below (it is, due to constraints).

Conclusion: Use 15 units of F1 and 7.5 units of F2 for minimum cost of ₹525 while meeting all nutritional requirements.

Note: Since we can't buy half units in reality, this would be an integer programming problem, requiring adjustment to integer values near (15, 7.5).

Example: Challenging Linear Programming Problem

Minimize z = 3x + 5y subject to: x + 2y ≥ 10, x + y ≥ 6, 3x + y ≥ 8, x ≥ 0, y ≥ 0.

Solution:

Step 1: Graph constraints (all ≥, so feasible region is unbounded above).

Step 2: Find corner points by solving systems:
• Intersection of x+2y=10 and x+y=6: Subtract ⇒ y=4, then x=2 ⇒ A(2,4)
• Intersection of x+2y=10 and 3x+y=8: Solve ⇒ x=1.2, y=4.4 ⇒ B(1.2, 4.4)
• Intersection of x+y=6 and 3x+y=8: Subtract ⇒ 2x=2 ⇒ x=1, then y=5 ⇒ C(1,5)
• Also check intercepts: x+2y=10 meets axes at (10,0) and (0,5); x+y=6 at (6,0) and (0,6); 3x+y=8 at (8/3,0) and (0,8)

Step 3: Feasible region is unbounded polygon with vertices A(2,4), B(1.2,4.4), C(1,5).

Step 4: Evaluate z = 3x + 5y at corner points:
• A(2,4): z = 3(2)+5(4)=6+20=26
• B(1.2,4.4): z = 3(1.2)+5(4.4)=3.6+22=25.6
• C(1,5): z = 3(1)+5(5)=3+25=28

Step 5: Minimum value appears to be 25.6 at B(1.2, 4.4).

Step 6: Check if minimum exists (feasible region is unbounded, but objective increases as we move away from origin in positive direction, so minimum exists at a corner point).

Step 7: Verify by checking points near B: Point (1,4.5): z=3+22.5=25.5 < 25.6. Need to check if (1,4.5) is feasible:
• x+2y=1+9=10 ≥ 10 ✓
• x+y=1+4.5=5.5 ≥ 6 ✗ (not feasible)
So (1,4.5) is not in feasible region.

Step 8: Minimum is indeed at B(1.2, 4.4) with z=25.6.

Conclusion: Minimum value is 25.6 at x=1.2, y=4.4.

Important Points for JEE/NEET

  • Linear inequality: ax + b < c, ax + b > c, ax + b ≤ c, or ax + b ≥ c
  • When multiplying/dividing by negative number: REVERSE the inequality sign
  • Solution of inequality in one variable: Usually an interval
  • Interval notation: ( ) for open intervals, [ ] for closed intervals
  • Graphical representation of ax + by < c: Half-plane below line ax + by = c (if b > 0)
  • Boundary line: Solid for ≤ or ≥, dashed for < or >
  • Test point method: Use (0,0) if not on boundary line to determine which half-plane to shade
  • System of linear inequalities: Solution is intersection of individual solution sets (feasible region)
  • Feasible region: Can be bounded (polygon) or unbounded
  • Linear programming: Optimize linear objective function subject to linear constraints
  • Fundamental Theorem of LP: If optimal solution exists, it occurs at a corner point of feasible region
  • Corner point method: Evaluate objective function at each vertex to find optimum
  • Multiple optimal solutions: If two corner points give same optimal value, all points on line segment between them are optimal
  • Applications: Production planning, diet problems, transportation problems, portfolio optimization
  • Formulating LP problems: Define variables, write objective function, write constraints

Common Mistakes to Avoid:

  • Forgetting to reverse inequality sign when multiplying/dividing by negative number
  • Using wrong line style (solid vs dashed) for boundary lines
  • Shading wrong half-plane in graphical method
  • Not including endpoints in interval notation when inequality includes equality
  • Mistaking intersection (∩) for union (∪) when solving systems of inequalities
  • Not considering non-negativity constraints (x ≥ 0, y ≥ 0) in real-world problems
  • Assuming optimal solution always exists (unbounded feasible region may have no optimum)
  • Not checking all corner points in linear programming problems
  • Incorrectly solving simultaneous equations to find corner points
  • Not converting word problems into proper mathematical formulation
  • Forgetting to check if test point (0,0) is on boundary line before using it
  • Not labeling graphs properly (axes, intercepts, shading)
  • Confusing maximization and minimization problems
  • Not considering integer solutions when variables represent countable items
