Introduction to HTML

HTML (HyperText Markup Language) is the standard markup language for creating web pages and web applications. It describes the structure of a web page semantically and originally included cues for the appearance of the document.

What is HTML?

HTML is the foundation of all web pages. It provides:

  • Structure: Defines headings, paragraphs, lists, and other content elements
  • Semantics: Gives meaning to web content for browsers and assistive technologies
  • Hyperlinks: Creates links between pages (the "HT" in HTML)
  • Multimedia: Embeds images, videos, and audio
  • Forms: Creates interactive elements for user input
HTML5: The current version is HTML5, which introduced new semantic elements, multimedia support, and improved APIs for web applications.

Your First HTML Document

The basic structure of an HTML document:

index.html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Web Page</title>
</head>
<body>
    <h1>Hello, World!</h1>
    <p>This is my first HTML page.</p>
</body>
</html>

Breaking this down:

  • <!DOCTYPE html> declares this is an HTML5 document
  • <html> is the root element that wraps all content
  • <head> contains metadata and links to external resources
  • <body> contains all visible page content
  • <h1> is a heading element (most important heading)
  • <p> is a paragraph element
Case Sensitivity: HTML is NOT case-sensitive for element names, but lowercase is standard practice. Attribute values can be case-sensitive depending on the attribute.

HTML Document Structure

Every valid HTML document follows a specific structure that browsers understand.

DOCTYPE Declaration

The DOCTYPE declaration tells the browser which version of HTML the page is using:

DOCTYPE examples
<!-- HTML5 -->
<!DOCTYPE html>

<!-- HTML 4.01 Strict -->
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" 
"http://www.w3.org/TR/html4/strict.dtd">

<!-- XHTML 1.0 Strict -->
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" 
"http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
Always use <!DOCTYPE html> for modern web development. It ensures browsers render your page in standards mode.

HTML Element

The <html> element is the root of an HTML document and should include the lang attribute:

html element
<html lang="en">
<!-- All content goes here -->
</html>

The lang attribute is important for:

  • Accessibility (screen readers use it to determine pronunciation)
  • Search engine optimization (helps with language targeting)
  • Browser features (like translation tools)

Head Section

The <head> contains metadata and resources not displayed on the page:

head section example
<head>
    <!-- Character encoding -->
    <meta charset="UTF-8">
    
    <!-- Viewport for responsive design -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    
    <!-- Page title (appears in browser tab) -->
    <title>Page Title</title>
    
    <!-- Link to CSS file -->
    <link rel="stylesheet" href="styles.css">
    
    <!-- Link to favicon -->
    <link rel="icon" href="favicon.ico" type="image/x-icon">
    
    <!-- Meta description for SEO -->
    <meta name="description" content="A brief description of the page">
    
    <!-- Link to JavaScript file (deferred) -->
    <script src="script.js" defer></script>
</head>

Body Section

The <body> contains all visible content:

body section structure
<body>
    <!-- Header section -->
    <header>
        <h1>Website Header</h1>
        <nav>
            <!-- Navigation links -->
        </nav>
    </header>
    
    <!-- Main content -->
    <main>
        <!-- Article content -->
    </main>
    
    <!-- Footer section -->
    <footer>
        <!-- Footer content -->
    </footer>
</body>

HTML5 Syntax: In HTML5, you can write void elements without the closing slash: <br> instead of <br />. Both are valid, but the shorter form is more common.

HTML Attributes

Attributes provide additional information about HTML elements. They are always specified in the start tag and usually come in name/value pairs: name="value"

Common Global Attributes

These attributes can be used on most HTML elements:

<!-- class: One or more class names -->
<p class="text-primary highlight">...</p>

<!-- style: Inline CSS -->
<h1 style="color: blue; font-size: 24px;">...</h1>

<!-- title: Tooltip text -->
<abbr title="HyperText Markup Language">HTML</abbr>

<!-- data-*: Custom data attributes -->
<div data-user-id="123" data-role="admin">...</div>

<!-- aria-*: Accessibility attributes -->
<button aria-label="Close menu">X</button>

<!-- tabindex: Tab navigation order -->
<button tabindex="1">First</button>
<button tabindex="2">Second</button>

Boolean attributes don't need a value. Their presence alone makes them true:

Boolean Attribute Values: While you can write required="required" or checked="checked", the shorter form without a value is preferred in HTML5.

Links and Images

Creating Links

The <a> (anchor) element creates hyperlinks:

links.html
<!-- Link to external website -->
<a href="https://www.example.com">Visit Example</a>

<!-- Link to same page (fragment identifier) -->
<a href="#section2">Go to Section 2</a>
<h2 id="section2">Section 2</h2>

<!-- Link to another page in same site -->
<a href="/about.html">About Us</a>

<!-- Link to email address -->
<a href="mailto:someone@example.com">Send Email</a>

<!-- Link to phone number -->
<a href="tel:+1234567890">Call Us</a>

<!-- Link to download a file -->
<a href="/files/document.pdf" download>Download PDF</a>

<!-- Link that opens in new tab -->
<a href="https://example.com" target="_blank" rel="noopener noreferrer">
    External Link (New Tab)
</a>
Security Note: Always use rel="noopener noreferrer" with target="_blank" to prevent security vulnerabilities.

Working with Images

The <img> element embeds images:

images.html
<!-- Basic image -->
<img src="image.jpg" alt="Description of image">

<!-- Image with dimensions -->
<img 
    src="photo.png"
    alt="A scenic view"
    width="800"
    height="600"
>

<!-- Responsive image with srcset -->
<img
    src="image-small.jpg"
    srcset="image-small.jpg 400w, image-medium.jpg 800w, image-large.jpg 1200w"
    sizes="(max-width: 600px) 400px, (max-width: 1200px) 800px, 1200px"
    alt="Responsive image"
    loading="lazy"
>

<!-- Figure with caption -->
<figure>
    <img src="chart.png" alt="Sales chart for Q1 2023">
    <figcaption>Figure 1: Quarterly Sales Data</figcaption>
</figure>

<!-- Image as link -->
<a href="large-image.jpg">
    <img src="thumbnail.jpg" alt="Click to enlarge">
</a>

Image Formats and Best Practices

  • JPEG/JPG: Best for photographs (supports millions of colors)
  • PNG: Best for graphics with transparency (lossless compression)
  • GIF: Simple animations and graphics with limited colors
  • WebP: Modern format with excellent compression (use with fallback)
  • SVG: Vector graphics that scale without quality loss
Always include alt text: The alt attribute is required for accessibility. Screen readers read it aloud, and it displays if the image fails to load.

Always Validate Server-Side: HTML5 validation is for user experience, but always validate data on the server as well. Client-side validation can be bypassed.

Semantic HTML

Semantic HTML uses HTML elements that clearly describe their meaning to both browser and developer. Semantic elements make HTML more readable and improve accessibility.

HTML5 Semantic Elements

semantic_layout.html
<body>
    <!-- Header: introductory content -->
    <header>
        <h1>Website Title</h1>
        <p>Website slogan or tagline</p>
        
        <!-- Navigation: major navigation blocks -->
        <nav>
            <ul>
                <li><a href="/">Home</a></li>
                <li><a href="/about">About</a></li>
                <li><a href="/contact">Contact</a></li>
            </ul>
        </nav>
    </header>
    
    <!-- Main: dominant content of the document -->
    <main>
        
        <!-- Article: independent, self-contained content -->
        <article>
            <header>
                <h2>Article Title</h2>
                <p>Posted on <time datetime="2023-10-05">October 5, 2023</time></p>
            </header>
            
            <!-- Section: thematic grouping of content -->
            <section>
                <h3>Introduction</h3>
                <p>First paragraph of the article...</p>
            </section>
            
            <section>
                <h3>Main Content</h3>
                <p>More content here...</p>
                
                <!-- Aside: tangentially related content -->
                <aside>
                    <h4>Related Information</h4>
                    <p>This is related to the main content.</p>
                </aside>
            </section>
            
            <footer>
                <p>Article footer with tags or author info.</p>
            </footer>
        </article>
        
    </main>
    
    <!-- Footer: for its nearest ancestor -->
    <footer>
        <p>© 2023 Company Name. All rights reserved.</p>
        <address>
            Contact: <a href="mailto:info@example.com">info@example.com</a>
        </address>
    </footer>
</body>

Benefits of Semantic HTML

  • Accessibility: Screen readers can better navigate the page
  • SEO: Search engines understand content better
  • Maintainability: Code is easier to read and understand
  • Future-proof: Semantic markup adapts better to new devices
  • Interoperability: Other systems can process content more easily

Common Semantic Elements

Element Purpose Example
<header> Introductory content Page header, article header
<nav> Navigation links Main menu, table of contents
<main> Dominant content Main content area
<article> Self-contained composition Blog post, news article
<section> Thematic grouping Chapter, tab content
<aside> Tangentially related Sidebar, pull quote
<footer> Closing content Page footer, article footer

HTML Accessibility

Accessible HTML ensures that people with disabilities can use your website.

Key Accessibility Principles

1. Semantic HTML

Use the right elements for the right purpose:

<!-- Good: Semantic button -->
<button onclick="submitForm()">Submit</button>

<!-- Bad: Non-semantic button -->
<div onclick="submitForm()">Submit</div>

2. ARIA Attributes

ARIA (Accessible Rich Internet Applications) enhances accessibility:

aria_examples.html
<!-- Button with ARIA label -->
<button aria-label="Close notification dialog">X</button>

<!-- Navigation with ARIA role -->
<nav role="navigation" aria-label="Main menu">
    <!-- Navigation links -->
</nav>

<!-- Live region for dynamic content -->
<div aria-live="polite" aria-atomic="true">
    <span id="message"></span>
</div>

<!-- Modal dialog -->

Accessibility Checklist

  • Use semantic HTML elements
  • Provide alternative text for images (alt attribute)
  • Ensure sufficient color contrast (minimum 4.5:1 for normal text)
  • Make all functionality available via keyboard
  • Use ARIA attributes when semantic HTML isn't enough
  • Provide clear focus indicators
  • Write descriptive link text (avoid "click here")
  • Use headings properly (h1 h6 hierarchy)
  • Ensure forms have associated labels
  • Make tables accessible with captions and headers
Testing Accessibility: Use tools like Lighthouse (in Chrome DevTools), axe DevTools, and screen readers (NVDA, VoiceOver, JAWS) to test your website's accessibility.

HTML Best Practices

Follow these guidelines to write clean, maintainable, and efficient HTML code.

1. Use Semantic HTML

<!-- Good: Semantic -->
<header>
<nav>
<main>
<article>
<section>
<footer>

<!-- Avoid: Non-semantic -->
<div id="header">
<div class="nav">
<div id="main">
<div class="article">
<div class="section">
<div id="footer">

2. Write Valid HTML

  • Always include DOCTYPE
  • Close all elements properly
  • Use lowercase for element and attribute names
  • Quote all attribute values
  • Use the HTML validator: W3C Validator

3. Optimize Performance

<!-- Lazy load images -->
<img src="image.jpg" alt="..." loading="lazy">

<!-- Async or defer scripts -->
<script src="script.js" defer></script>

<!-- Preload critical resources -->
<link rel="preload" href="style.css" as="style">

4. Maintain Accessibility

<!-- Always include alt text -->
<img src="photo.jpg" alt="Two people hiking in mountains">

<!-- Use proper heading hierarchy -->
<h1>Page Title</h1>
<h2>Section Title</h2>
<h3>Subsection Title</h3>

<!-- Label all form elements -->
<label for="email">Email Address:</label>
<input type="email" id="email" name="email">

5. Keep It Simple and Readable

  • Use consistent indentation (2 or 4 spaces)
  • Add comments for complex sections
  • Avoid inline styles (use CSS classes instead)
  • Keep nesting shallow (avoid deeply nested elements)
  • Use meaningful class and ID names

6. Security Considerations

<!-- Prevent security issues with target="_blank" -->
<a href="https://external.com" target="_blank" rel="noopener noreferrer">
    External Link
</a>

<!-- Sanitize user input (prevent XSS) -->
<!-- Never trust user input in innerHTML or document.write() -->

<!-- Use HTTPS for all resources -->
<script src="https://cdn.example.com/library.js"></script>
Progressive Enhancement: Build core functionality with HTML, enhance with CSS, and add interactivity with JavaScript. This ensures your site works even if CSS or JavaScript fails to load.
