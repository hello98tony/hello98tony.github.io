Introduction to Permutations & Combinations

Permutations and Combinations are fundamental concepts in combinatorics that deal with counting and arranging objects. These concepts form the basis for probability theory and have numerous real-world applications.

Key Difference: Permutations are about arrangements (order matters), while Combinations are about selections (order doesn't matter).

Permutations

Counting the number of ways to arrange objects in a specific order. ABC is different from BAC.

ⁿPᵣ = n!/(n-r)!
Combinations

Counting the number of ways to select objects without regard to order. ABC is same as BAC.

ⁿCᵣ = n!/[r!(n-r)!]

Fundamental Principles of Counting

These principles form the foundation of all counting problems. Understanding them is crucial for solving complex permutation and combination problems.

Multiplication Principle

If one operation can be done in m ways and another independent operation in n ways, then both operations together can be done in m×n ways.

Total ways = m × n

Example: If there are 3 shirts and 4 pants, number of outfits = 3 × 4 = 12

Addition Principle

If one operation can be done in m ways and another in n ways (mutually exclusive), then either operation can be done in m + n ways.

Total ways = m + n

Example: Travel by bus (3 routes) or train (2 routes): Total ways = 3 + 2 = 5

Inclusion-Exclusion Principle

For counting elements in union of sets: |A ∪ B| = |A| + |B| - |A ∩ B|

|A ∪ B| = |A| + |B| - |A ∩ B|

Example: In a class, 20 students play cricket, 15 play football, 8 play both. Total players = 20 + 15 - 8 = 27

When to Use Which Principle:

  • Multiplication: When operations are performed in sequence (AND situation)
  • Addition: When operations are alternatives (OR situation)
  • Both: Many problems require using both principles together
Example: Combined Principles

A restaurant offers 4 appetizers, 6 main courses, and 3 desserts. A meal consists of either an appetizer and main course, or a main course and dessert. How many different meals are possible?

Solution:

Case 1: Appetizer and main course = 4 × 6 = 24

Case 2: Main course and dessert = 6 × 3 = 18

Total meals = 24 + 18 = 42 (using both multiplication and addition principles)

Factorial Notation

Factorial is a fundamental concept in permutations and combinations. It represents the product of all positive integers up to a given number.

Factorial (n!): The product of all positive integers from 1 to n. Defined as n! = n × (n-1) × (n-2) × ... × 3 × 2 × 1

n! = n × (n-1) × (n-2) × ... × 3 × 2 × 1

Permutations

A permutation is an arrangement of objects in a specific order. The number of permutations depends on whether objects are distinct or identical, and whether repetition is allowed.

Permutation (ⁿPᵣ): The number of ways to arrange r objects from n distinct objects, where order matters.

ⁿPᵣ = n!/(n-r)!

Where:
• n = total number of distinct objects
• r = number of objects being arranged
• n ≥ r ≥ 0

Permutation of all n objects
ⁿPₙ = n!

Arranging all n distinct objects

Permutation with repetition
nʳ

Each of r positions has n choices

Circular Permutation
(n-1)!

Arrangements around a circle (rotations considered same)

Types of Permutations

Type Formula When to Use Example

With repetition nʳ Each position has n choices 3-digit numbers from 0-9
Circular (n-1)! Arranging around circle Seating 6 at round table

Special Cases with Examples

Permutations with Repetition

When repetition is allowed, each position has n choices.

Number of arrangements = nʳ

Example: Number of 3-letter words using A,B,C with repetition allowed = 3³ = 27

Permutations of Alike Objects

When some objects are identical, we divide by factorial of identical objects.

n!/(p!q!r!...)

Example: Arrange letters of "MISSISSIPPI": 11!/(4!4!2!)

Circular Permutations

In circular arrangements, rotations are considered same.

(n-1)! for distinct objects

Example: 5 people at round table: (5-1)! = 4! = 24 arrangements

Key Formulas Summary:

  • Linear permutation (distinct): ⁿPᵣ = n!/(n-r)!
  • Linear permutation (all): n!
  • With repetition allowed: nʳ
  • With p alike of one kind, q alike of another: n!/(p!q!)
  • Circular permutation: (n-1)!
  • Necklace/bracelet (flippable): (n-1)!/2
Example: Permutation with Restrictions

How many ways can 5 boys and 3 girls be arranged in a row if no two girls sit together?

Solution:

First arrange boys: 5! = 120 ways

This creates 6 gaps (before, between, after boys): _B_B_B_B_B_

Choose 3 gaps for girls: ⁶P₃ = 6×5×4 = 120 ways

Arrange girls in chosen gaps: 3! = 6 ways

Total = 120 × 120 × 6 = 86,400 ways

Derangements

Arrangements where no object is in its original position.

!n = n!∑(k=0 to n) (-1)ᵏ/k!

For n=3: !3 = 3!(1 - 1 + 1/2 - 1/6) = 6×(1/3) = 2

For n=4: !4 = 9

Also: !n = (n-1)[!(n-1) + !(n-2)]

Combinations

A combination is a selection of objects without regard to order. It answers the question: "In how many ways can we choose r objects from n objects?"

Combination (ⁿCᵣ): The number of ways to select r objects from n distinct objects, where order does not matter.

ⁿCᵣ = n!/[r!(n-r)!]

Where:
• n = total number of distinct objects
• r = number of objects being selected
• n ≥ r ≥ 0

Basic Formula
ⁿCᵣ = n!/[r!(n-r)!]

Also written as C(n,r) or (ⁿᵣ)

Complementary
ⁿCᵣ = ⁿCₙ₋ᵣ

Choosing r is same as rejecting n-r

Pascal's Identity
ⁿCᵣ + ⁿCᵣ₋₁ = ⁿ⁺¹Cᵣ

Forms Pascal's Triangle

Important Properties of Combinations

Property Formula Explanation
Complementary ⁿCᵣ = ⁿCₙ₋ᵣ Choosing r = rejecting n-r
Pascal's Identity ⁿCᵣ + ⁿCᵣ₋₁ = ⁿ⁺¹Cᵣ Recursive relation
Sum of row in Pascal's Triangle ⁿC₀ + ⁿC₁ + ... + ⁿCₙ = 2ⁿ All subsets of n elements
Alternating sum ⁿC₀ - ⁿC₁ + ⁿC₂ - ... = 0 For n ≥ 1
Vandermonde's Identity ∑(ᵐCₖ × ⁿCᵣ₋ₖ) = ᵐ⁺ⁿCᵣ Where k goes from 0 to r

Key Relations:

  • ⁿC₀ = ⁿCₙ = 1 (empty set and full set)
  • ⁿC₁ = n (selecting 1 from n)
  • ⁿCᵣ = ⁿCₙ₋ᵣ (symmetry)
  • ⁿCᵣ = ⁿ⁻¹Cᵣ + ⁿ⁻¹Cᵣ₋₁ (Pascal's rule)
  • r × ⁿCᵣ = n × ⁿ⁻¹Cᵣ₋₁ (useful for simplification)

Special Types of Combinations

Combinations with Repetition

When objects can be chosen more than once (multisets).

Number of ways = ⁿ⁺ʳ⁻¹Cᵣ

Example: Choose 3 fruits from 5 types with repetition: ⁵⁺³⁻¹C₃ = ⁷C₃ = 35

Combinations with Restrictions

When certain objects must be included or excluded.

Include k specific objects: ⁿ⁻ᵏCᵣ₋ₖ

Exclude k specific objects: ⁿ⁻ᵏCᵣ

Example: Choose 4 from 10 including A: ⁹C₃ = 84

Division into Groups

Dividing n distinct objects into groups of sizes r₁, r₂, ..., rₖ.

n!/(r₁!r₂!...rₖ!)

Example: Divide 10 into groups of 4,3,3: 10!/(4!3!3!)

Example: Combination Problem

In how many ways can a committee of 5 be chosen from 7 men and 5 women if it must contain at least 3 men?

Solution:

Case 1: 3 men, 2 women = ⁷C₃ × ⁵C₂ = 35 × 10 = 350

Case 2: 4 men, 1 woman = ⁷C₄ × ⁵C₁ = 35 × 5 = 175

Case 3: 5 men, 0 women = ⁷C₅ × ⁵C₀ = 21 × 1 = 21

Total = 350 + 175 + 21 = 546 ways

Probability Applications

Permutations and combinations are fundamental to calculating probabilities. The basic probability formula:

P(event) = Number of favorable outcomes / Total possible outcomes

Common Probability Problems:

  • Birthday Problem: Probability that 2 people in room share birthday
  • Committee Problems: Probability committee has certain composition
  • Card Problems: Probability of specific poker hands
  • Coin/Dice: Probability of specific outcomes in repeated trials
  • Sampling: Probability in quality control sampling
Example: Probability Application

A bag contains 5 red, 4 blue, and 3 green balls. If 3 balls are drawn at random, what is the probability that:

a) All are red?
b) Exactly 2 are blue?
c) At least 1 is green?

Solution:

Total outcomes = ¹²C₃ = 220

a) P(all red) = ⁵C₃/¹²C₃ = 10/220 = 1/22

b) P(2 blue, 1 other) = (⁴C₂ × ⁸C₁)/¹²C₃ = (6 × 8)/220 = 48/220 = 12/55

c) P(at least 1 green) = 1 - P(no green) = 1 - ⁹C₃/¹²C₃ = 1 - 84/220 = 136/220 = 34/55

Problem Solving Strategies

Effective techniques for solving permutation and combination problems in JEE and other competitive exams.

Type 1: Basic Counting

Steps:

  1. Identify if order matters (Permutation) or not (Combination)
  2. Check for restrictions (must include/exclude, together/apart)
  3. Apply appropriate formula
  4. Multiply/Add for multiple cases

Tip: If unsure, consider small example to test logic

Type 2: Restrictions

Steps:

  1. Treat restricted objects as units if they must be together
  2. Use gap method if objects must be separate
  3. For "at least" problems, use complement method
  4. Break into cases if needed

Tip: For circular arrangements, fix one position first

Type 3: Probability

Steps:

  1. Calculate total possible outcomes
  2. Calculate favorable outcomes
  3. Divide: P = favorable/total
  4. Check if order matters in both numerator and denominator

Tip: Use combinations for card problems, permutations for arrangements

Common Problem Types:

  • Seating Arrangements: People in row/circle, restrictions
  • Word Formation: Letters with/without repetition, restrictions
  • Committee Selection: With/without restrictions, men/women
  • Number Formation: Digits with/without repetition, divisibility
  • Distribution: Identical/distinct objects to persons/boxes
  • Geometry: Points, lines, triangles, polygons

Advanced Problem Solving

• For non-negative integer solutions to x₁ + x₂ + ... + xₖ = n: ⁿ⁺ᵏ⁻¹Cₖ₋₁

• For positive integer solutions to x₁ + x₂ + ... + xₖ = n: ⁿ⁻¹Cₖ₋₁

• Derangements: !n = n!∑(-1)ᵏ/k!

• Principle of Inclusion-Exclusion for complex counting

• Generating functions for sequence counting

• Recurrence relations for complex arrangements

Example: Complex Problem

Find number of ways to arrange letters of "ENGINEERING" such that all vowels are together and all consonants are together.

Solution:

Vowels: E, I, E, E, I (5 vowels with E repeated 3 times, I repeated 2 times)

Consonants: N, G, N, R, N, G (6 consonants with N repeated 3 times, G repeated 2 times, R once)

Treat vowel group as 1 unit, consonant group as 1 unit: 2! = 2 arrangements

Arrange vowels within group: 5!/(3!2!) = 10 ways

Arrange consonants within group: 6!/(3!2!1!) = 60 ways

Total = 2 × 10 × 60 = 1200 ways

Important Points for JEE/NEET

  • Permutation: Order matters, Combination: Order doesn't matter
  • ⁿPᵣ = ⁿCᵣ × r! (multiply combinations by r! to get permutations)
  • For circular arrangements, fix one object to eliminate rotations
  • For "at least" problems, use complement: P(at least 1) = 1 - P(none)
  • In combinations with repetition, formula is ⁿ⁺ʳ⁻¹Cᵣ
  • Number of diagonals in n-sided polygon = ⁿC₂ - n
  • Number of handshakes among n people = ⁿC₂
  • When dividing into groups, divide by factorial if groups are identical
  • For non-negative integer solutions to x₁+...+xₖ=n: ⁿ⁺ᵏ⁻¹Cₖ₋₁
  • For positive integer solutions to x₁+...+xₖ=n: ⁿ⁻¹Cₖ₋₁

Common Mistakes to Avoid:

  • Confusing permutations and combinations
  • Forgetting to divide by factorial for identical objects
  • Not using complement method for "at least" problems
  • Incorrectly applying circular permutation formula
  • Miscounting gaps in "separate" problems
  • Using permutations when combinations are needed in probability
  • Forgetting that 0! = 1
  • Not considering all cases in complex problems
  • Incorrect application of inclusion-exclusion principle
  • Mistaking "arrange" (permutation) vs "select" (combination)
